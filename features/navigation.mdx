## Overview

BioTea uses [Expo Router](https://docs.expo.dev/router/introduction/) for navigation, which provides file-based routing similar to Next.js. The app structure includes Stack navigation at the root level and Tab navigation for the main authenticated screens.

## File-Based Routing

Expo Router automatically creates routes based on your file structure. Each file in the `app` directory becomes a route:

```
app/
├── _layout.tsx          → Root layout (Stack navigator)
├── index.tsx            → / (landing page)
├── login.tsx            → /login
├── register.tsx         → /register
└── (tabs)/              → /(tabs) (tab group)
    ├── _layout.tsx      → Tab navigator
    ├── (home)/          → Home tab
    │   ├── _layout.tsx  → Home stack
    │   └── inic.tsx     → /(tabs)/(home)/inic
    └── (settings)/      → Settings tab
        ├── _layout.tsx  → Settings stack
        ├── settingsG.tsx
        ├── general.tsx
        ├── notification.tsx
        └── user.tsx
```

<Note>
Folders wrapped in parentheses like `(tabs)`, `(home)`, and `(settings)` are route groups. They organize files without adding segments to the URL path.
</Note>

## Root Layout

The root layout in `app/_layout.tsx:1` configures the main Stack navigator and context providers:

```typescript app/_layout.tsx
import { Stack } from "expo-router";
import { UserProvider } from "../components/UserContext";
import { SecureStoreProvider } from "../components/SecureStoreContext";

export default function RootLayout() {
  return (
    <UserProvider>
      <SecureStoreProvider>
        <Stack
          screenOptions={{
            headerStyle: {
              backgroundColor: '#f4511e',
            },
            headerTintColor: '#fff',
            headerTitleStyle: {
              fontWeight: 'bold',
            },
          }}
        >
          <Stack.Screen
            name="index"
            options={{
              title: 'BioTea App',
              headerShown: true
            }}
          />
          <Stack.Screen name="login" />
          <Stack.Screen name="register" />
          <Stack.Screen
            name="(tabs)"
            options={{
              headerShown: false
            }}
          />
        </Stack>
      </SecureStoreProvider>
    </UserProvider>
  );
}
```

## Tab Navigation

The tab navigator is defined in `app/(tabs)/_layout.tsx:1`:

```typescript app/(tabs)/_layout.tsx
import FontAwesome from '@expo/vector-icons/FontAwesome';
import { Tabs } from 'expo-router';

export default function TabLayout() {
    return (
        <Tabs screenOptions={{ tabBarActiveTintColor: 'blue' }}>
            <Tabs.Screen
                name="(home)"
                options={{
                    title: 'Home',
                    tabBarIcon: ({ color }) => <FontAwesome size={28} name="home" color={color} />,
```

## Navigation Methods

Expo Router provides several ways to navigate between screens:

<CodeGroup>
```typescript useRouter Hook
import { useRouter } from 'expo-router';

export default function MyScreen() {
    const router = useRouter();
    
    // Navigate to a route
    const handleLogin = () => {
        router.navigate('(tabs)/(home)/inic');
    };
    
    // Go back
    const handleBack = () => {
        router.back();
    };
    
    // Replace current route
    const handleReplace = () => {
        router.replace('/login');
    };
}
```

```typescript Link Component
import { Link } from 'expo-router';

export default function MyScreen() {
    return (
        <Link href="/(tabs)/(home)/inic">
            <Text>Go to Home</Text>
        </Link>
    );
}
```

```typescript Programmatic Navigation (app/login.tsx:38)
import { useRouter } from 'expo-router';

export default function Login() {
    const router = useRouter();
    
    const handleLogin = async () => {
        // ... authentication logic
        
        if (!response.data.error) {
            setUser(response.data.data);
            await saveSecureData('user', response.data.data);
            
            // Navigate to home after successful login
            router.navigate('(tabs)/(home)/inic');
        }
    };
}
```
</CodeGroup>

## Common Navigation Patterns

### Login Flow

After successful authentication, navigate to the main app:

```typescript
// From login screen to home
router.navigate('(tabs)/(home)/inic');
```

### Logout Flow

Clear user data and return to login:

```typescript
const handleLogout = async () => {
    // Clear state
    setUser(null);
    await deleteSecureData('user');
    
    // Navigate to login
    router.replace('/login');
};
```

<Note>
Use `router.replace()` for logout to prevent users from navigating back to authenticated screens.
</Note>

### Deep Linking

Navigate to specific nested routes:

```typescript
// Navigate to settings → user profile
router.navigate('(tabs)/(settings)/user');

// Navigate to settings → notifications
router.navigate('(tabs)/(settings)/notification');
```

## Navigation Guards

Protect authenticated routes by checking user state:

```typescript
import { useEffect } from 'react';
import { useRouter } from 'expo-router';
import { useUser } from '../components/UserContext';

export default function ProtectedScreen() {
    const { user } = useUser();
    const router = useRouter();
    
    useEffect(() => {
        if (!user) {
            // Redirect to login if not authenticated
            router.replace('/login');
        }
    }, [user]);
    
    if (!user) return null;
    
    return <View>{/* Protected content */}</View>;
}
```

## Best Practices

<Warning>
**Navigation Best Practices:**

1. Use `router.replace()` for authentication flows to prevent back navigation
2. Use `router.navigate()` for standard navigation to maintain history
3. Always check user authentication before rendering protected screens
4. Use typed route parameters with TypeScript for type safety
5. Keep navigation logic close to user actions (button handlers)
6. Use route groups `()` to organize files without affecting URLs
</Warning>
