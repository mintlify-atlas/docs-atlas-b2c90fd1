## Overview

BioTea uses Expo's SecureStore API to securely persist sensitive data like user credentials and authentication tokens. The `SecureStoreContext` provides a React Context wrapper around SecureStore, making it easy to save, load, and delete encrypted data throughout the app.

## Why Secure Storage?

SecureStore provides encrypted storage on both iOS and Android:

- **iOS**: Uses the Keychain Services
- **Android**: Uses SharedPreferences with encryption
- **Web**: Not supported (use alternative storage)

<Warning>
Never store sensitive data (passwords, tokens, personal information) in AsyncStorage or plain text. Always use SecureStore for sensitive information.
</Warning>

## SecureStoreContext Implementation

The complete implementation is in `components/SecureStoreContext.tsx:1`:

```typescript components/SecureStoreContext.tsx
import React, { createContext, useContext, ReactNode } from "react";
import * as SecureStore from "expo-secure-store";

interface SecureStoreContextType {
    saveSecureData: (key: string, value: any) => Promise<void>;
    loadSecureData: (key: string) => Promise<any>;
    deleteSecureData: (key: string) => Promise<void>;
}

const SecureStoreContext = createContext<SecureStoreContextType | undefined>(undefined);

export function SecureStoreProvider({ children }: { children: ReactNode }) {
    const saveSecureData = async (key: string, value: any) => {
        await SecureStore.setItemAsync(key, JSON.stringify(value));
    };

    const loadSecureData = async (key: string) => {
        const data = await SecureStore.getItemAsync(key);
        return data ? JSON.parse(data) : null;
    };

    const deleteSecureData = async (key: string) => {
        await SecureStore.deleteItemAsync(key);
    };

    return (
        <SecureStoreContext.Provider
            value={{ saveSecureData, loadSecureData, deleteSecureData }}
        >
            {children}
        </SecureStoreContext.Provider>
    );
}

export function useSecureStore() {
    const context = useContext(SecureStoreContext);
    if (context === undefined) {
        throw new Error("useSecureStore must be used within a SecureStoreProvider");
    }
    return context;
}
```

## API Methods

### saveSecureData

Saves data to encrypted storage. Automatically serializes objects to JSON.

```typescript
const saveSecureData = async (key: string, value: any) => {
    await SecureStore.setItemAsync(key, JSON.stringify(value));
};
```

**Parameters:**
- `key`: Unique identifier for the stored data
- `value`: Any serializable value (string, number, object, array)

**Example:**
```typescript
const { saveSecureData } = useSecureStore();

// Save user object
await saveSecureData('user', { nombres: 'John', correo: 'john@example.com' });

// Save authentication token
await saveSecureData('authToken', 'abc123xyz');
```

### loadSecureData

Retrrieves and deserializes data from encrypted storage.

```typescript
const loadSecureData = async (key: string) => {
    const data = await SecureStore.getItemAsync(key);
    return data ? JSON.parse(data) : null;
};
```

**Parameters:**
- `key`: The identifier for the stored data

**Returns:**
- The stored value (parsed from JSON)
- `null` if the key doesn't exist

**Example:**
```typescript
const { loadSecureData } = useSecureStore();

// Load user object
const user = await loadSecureData('user');
if (user) {
    console.log('User:', user.nombres);
}

// Load auth token
const token = await loadSecureData('authToken');
```

### deleteSecureData

Removes data from encrypted storage.

```typescript
const deleteSecureData = async (key: string) => {
    await SecureStore.deleteItemAsync(key);
};
```

**Parameters:**
- `key`: The identifier for the data to delete

**Example:**
```typescript
const { deleteSecureData } = useSecureStore();

// Clear user data on logout
await deleteSecureData('user');
await deleteSecureData('authToken');
```

## JSON Serialization

SecureStore only accepts string values, so the context automatically handles JSON serialization:

<Accordion title="How Serialization Works">
**Saving:**
```typescript
// You pass any object
await saveSecureData('user', { nombres: 'John', correo: 'john@example.com' });

// Internally converted to JSON string
await SecureStore.setItemAsync(key, JSON.stringify(value));
```

**Loading:**
```typescript
// Raw string retrieved from storage
const data = await SecureStore.getItemAsync(key);

// Automatically parsed back to object
return data ? JSON.parse(data) : null;
```
</Accordion>

<Note>
Make sure the data you store is JSON-serializable. Functions, symbols, and circular references cannot be stored.
</Note>

```typescript

    try {
        await saveSecureData('user', userData);

    }
};
```

## Storage Limits

<Warning>
**SecureStore Limitations:**

- Maximum value size: 2048 bytes on iOS
- Keys must be alphanumeric (can include `.`, `-`, and `_`)
- Total storage depends on device keychain limits
- Not available on web platform

For larger data, consider encrypting and using AsyncStorage or a local database.
</Warning>

## Best Practices

1. **Use Descriptive Keys**: Use clear, consistent key names (`'user'`, `'authToken'`, not `'u'`, `'t'`)

2. **Clean Up on Logout**: Always delete secure data when users log out
   ```typescript
   await deleteSecureData('user');
   await deleteSecureData('authToken');
   ```

3. **Sync with Context**: Keep SecureStore in sync with UserContext
   ```typescript
   // Update both
   setUser(userData);
   await saveSecureData('user', userData);
   ```

4. **Handle Null Cases**: Always check if loaded data exists
   ```typescript
   const user = await loadSecureData('user');
   if (user) {
       // Use user data
   }
   ```

5. **Validate Loaded Data**: Ensure loaded data is still valid
   ```typescript
   const token = await loadSecureData('authToken');
   if (token && isTokenValid(token)) {
       // Use token
   } else {
       await deleteSecureData('authToken');
   }
   ```
