<div className="mt-16 mb-16 max-w-5xl mx-auto px-6">
  <h2 className="text-2xl sm:text-3xl font-semibold text-gray-900 dark:text-white mb-4">
    Quick start
  </h2>
  <p className="text-base text-gray-600 dark:text-gray-400 mb-8">
    Get your development environment set up and run BioTea in minutes
  </p>
  
  <Steps>
    <Step title="Install dependencies">
      Clone the repository and install all required packages using npm.
      
      ```bash
      git clone https://github.com/Erick-Zam/BioTea.git
      cd BioTea
      npm install
      ```
    </Step>
    
    <Step title="Start the development server">
      Launch the Expo development server to run the app on your device or emulator.
      
      ```bash
      npx expo start
      ```
      
      <Note>
        You can press `a` for Android, `i` for iOS, or scan the QR code with the Expo Go app on your mobile device.
      </Note>
    </Step>
    
    <Step title="Configure API endpoint">
      The app connects to a backend API. Review and update the API endpoints in `constants/Urls.ts` if needed.
      
      ```typescript constants/Urls.ts
      export const apiEndpoints = {
        prod: 'https://simulacion7sb.000webhostapp.com/api',
        dev: 'https://simulacion7sb.000webhostapp.com/api',
        produccion: 'https://simulacion7sb.000webhostapp.com'
      };
      ```
    </Step>
    
    <Step title="Test authentication">
      Navigate to the login screen and test the authentication flow. User credentials are managed through the backend API.
      
      <Accordion title="Example login flow">
        When you submit credentials, the app sends a POST request to `/login.php` and stores the user data securely using Expo SecureStore. The response includes user information like name and email.
      </Accordion>
    </Step>
  </Steps>
</div>
