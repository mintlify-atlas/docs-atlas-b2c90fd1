## Prerequisites

Before you begin, ensure you have the following installed:

- Node.js (v16 or later)
- npm or yarn
- Expo CLI
- iOS Simulator (Mac only) or Android Emulator
- Expo Go app on your physical device (optional)

## Get Started

<Steps>
  <Step title="Clone the repository">
    Clone the BioTea repository to your local machine:

    ```bash
    git clone <repository-url>
    cd biotea
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using npm:

    ```bash
    npm install
    ```

    <Note>
      BioTea uses Expo SDK 51.0.8 with React Native 0.74.1. All dependencies will be installed automatically.
    </Note>

    Key dependencies include:
    - `expo-router` ~3.5.14 - File-based navigation
    - `expo-secure-store` ~13.0.1 - Secure credential storage
    - `axios` ^1.7.2 - HTTP client for API calls
    - `react-native-paper` ^5.12.3 - Material Design components
  </Step>

  <Step title="Start the development server">
    Launch the Expo development server:

    <CodeGroup>
      ```bash npm
      npm start
      ```

      ```bash Expo CLI
      npx expo start
      ```
    </CodeGroup>

    You'll see a QR code in your terminal and several options to open the app.
  </Step>

  <Step title="Open the app on your device">
    Choose one of the following methods:

    <Tip>
      For the best development experience, use a physical device or simulator rather than Expo Go.
    </Tip>

    **Option 1: Physical Device**
    - Install Expo Go from the App Store (iOS) or Google Play (Android)
    - Scan the QR code with your camera (iOS) or Expo Go app (Android)

    **Option 2: iOS Simulator** (Mac only)
    ```bash
    npm run ios
    ```

    **Option 3: Android Emulator**
    ```bash
    npm run android
    ```
  </Step>

  <Step title="Explore the app">
    The app will start on the welcome screen. From here you can:

    1. Navigate to the login screen
    2. Create an account or log in with existing credentials
    3. Access the home tab with user information
    4. Configure settings in the settings tab

    <Warning>
      The app requires an active backend connection. Ensure the API endpoints in `constants/Urls.ts` are accessible.
    </Warning>
  </Step>
</Steps>

## Project Structure

BioTea uses Expo Router for file-based navigation. Here's the key structure:

```
biotea/
├── app/
│   ├── _layout.tsx          # Root layout with providers
│   ├── index.tsx             # Welcome screen
│   ├── login.tsx             # Login screen
│   ├── register.tsx          # Registration screen
│   └── (tabs)/               # Tab navigation
│       ├── _layout.tsx       # Tab bar configuration
│       ├── (home)/           # Home tab screens
│       └── (settings)/       # Settings tab screens
├── components/
│   ├── UserContext.tsx       # User state management
│   ├── SecureStoreContext.tsx # Secure storage utilities
│   └── ...
├── constants/
│   ├── Urls.ts              # API endpoint configuration
│   └── Colors.ts            # Theme colors
└── package.json
```

## Understanding the Core Components

### Root Layout

The app wraps all screens with context providers for user management and secure storage:

```tsx app/_layout.tsx
import { Stack } from "expo-router";
import { UserProvider } from "../components/UserContext";
import { SecureStoreProvider } from "../components/SecureStoreContext";

export default function RootLayout() {
  return (
    <UserProvider>
      <SecureStoreProvider>
        <Stack>
          <Stack.Screen name="index" options={{ title: 'BioTea App' }} />
          <Stack.Screen name="login" />
          <Stack.Screen name="register" />
          <Stack.Screen name="(tabs)" options={{ headerShown: false }} />
        </Stack>
      </SecureStoreProvider>
    </UserProvider>
  );
}
```

### Tab Navigation

BioTea includes a bottom tab navigation with Home and Settings:

```tsx app/(tabs)/_layout.tsx
import FontAwesome from '@expo/vector-icons/FontAwesome';
import { Tabs } from 'expo-router';

export default function TabLayout() {
  return (
    <Tabs screenOptions={{ tabBarActiveTintColor: 'blue' }}>
      <Tabs.Screen
        name="(home)"
        options={{
          title: 'Home',
          tabBarIcon: ({ color }) => <FontAwesome size={28} name="home" color={color} />,
        }}
      />
      <Tabs.Screen
        name="(settings)"
        options={{
          title: 'Settings',
          tabBarIcon: ({ color }) => <FontAwesome size={28} name="cog" color={color} />,
        }}
      />
    </Tabs>
  );
}
```

### Authentication Example

BioTea uses axios for API calls and expo-secure-store for credential management:

```tsx app/login.tsx
import { useState } from "react";
import { useRouter } from 'expo-router';
import axios from "axios";
import { apiEndpoints } from "../constants/Urls";
import { useUser } from '../components/UserContext';
import { useSecureStore } from '../components/SecureStoreContext';

export default function Login() {
  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');
  const router = useRouter();
  const { setUser } = useUser();
  const { saveSecureData } = useSecureStore();

  const handleLogin = async () => {
    const response = await axios.post(
      `${apiEndpoints.dev}/login.php`,
      { user: username, pass: password }
    );

    if (!response.data.error) {
      setUser(response.data.data);
      await saveSecureData('user', response.data.data);
      router.navigate('(tabs)/(home)/inic');
    }
  };

  // ... UI implementation
}
```

## Configuration

### API Endpoints

Configure your backend URLs in `constants/Urls.ts`:

```ts constants/Urls.ts
export const apiEndpoints = {
  prod: 'https://simulacion7sb.000webhostapp.com/api',
  dev: 'https://simulacion7sb.000webhostapp.com/api',
  produccion: 'https://simulacion7sb.000webhostapp.com'
};
```

### App Configuration

Key settings are defined in `app.json`:

```json app.json
{
  "expo": {
    "name": "BioTea",
    "slug": "BioTea",
    "version": "1.0.0",
    "plugins": [
      "expo-router",
      "expo-secure-store"
    ],
    "experiments": {
      "typedRoutes": true
    }
  }
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Module not found errors">
    Clear the Metro bundler cache and reinstall dependencies:
    ```bash
    npm start -- --clear
    ```
    or
    ```bash
    rm -rf node_modules
    npm install
    ```
  </Accordion>

  <Accordion title="iOS build fails">
    If you're on a Mac and iOS builds fail, try:
    ```bash
    cd ios
    pod install
    cd ..
    npm run ios
    ```
  </Accordion>

  <Accordion title="Android emulator not detected">
    Ensure Android Studio is installed and an AVD (Android Virtual Device) is created and running before executing `npm run android`.
  </Accordion>

  <Accordion title="API connection issues">
    Verify that the API endpoints in `constants/Urls.ts` are accessible. Test the endpoints directly in a browser or with curl to ensure they're responding.
  </Accordion>
</AccordionGroup>
