## Project Structure Overview

The BioTea application follows a well-organized directory structure that separates concerns and makes the codebase maintainable and scalable.

```
BioTea/
├── app/                    # Application screens and routing
├── assets/                 # Static assets (images, fonts)
├── components/             # Reusable UI components
├── constants/              # Application constants
├── hooks/                  # Custom React hooks
├── scripts/                # Build and utility scripts
├── app.json               # Expo configuration
├── package.json           # Dependencies and scripts
├── tsconfig.json          # TypeScript configuration
└── babel.config.js        # Babel configuration
```

## Directory Breakdown

### `/app` - Application Routes

The `app/` directory contains all screen components and defines the navigation structure using expo-router's file-based routing system.

<Accordion title="Root Level Routes">
  **Location**: `app/`
  
  Core application entry points and authentication screens:
  
  - **`_layout.tsx`** - Root layout component that wraps the entire app with context providers (UserProvider, SecureStoreProvider) and configures the Stack navigator with custom header styling
  - **`index.tsx`** - Welcome/landing screen with navigation to login
  - **`login.tsx`** - User authentication screen with username/password inputs and API integration
  - **`register.tsx`** - User registration screen (implementation in progress)
  
  The root `_layout.tsx` establishes the foundational navigation structure:
  ```tsx
  <UserProvider>
    <SecureStoreProvider>
      <Stack>
        <Stack.Screen name="index" />
        <Stack.Screen name="login" />
        <Stack.Screen name="register" />
        <Stack.Screen name="(tabs)" />
      </Stack>
    </SecureStoreProvider>
  </UserProvider>
  ```
</Accordion>

<Accordion title="/(tabs) - Tab Navigation Group">
  **Location**: `app/(tabs)/`
  
  This route group implements the main tab navigation interface.
  
  - **`_layout.tsx`** - Tab navigator configuration with two tabs:
    - Home tab (using FontAwesome 'home' icon)
    - Settings tab (using FontAwesome 'cog' icon)
  
  **Nested Route Groups**:
  
  1. **`(home)/`** - Home tab stack
     - `_layout.tsx` - Stack navigator for home screens
     - `inic.tsx` - Main home screen (authenticated user landing page)
  
  2. **`(settings)/`** - Settings tab stack
     - `_layout.tsx` - Stack navigator for settings screens
     - `settingsG.tsx` - Main settings screen
     - `general.tsx` - General settings
     - `notification.tsx` - Notification preferences
     - `user.tsx` - User profile settings
</Accordion>

<Info>
  The parentheses `()` in folder names like `(tabs)`, `(home)`, and `(settings)` indicate **route groups** in expo-router. These groups organize routes without affecting the URL path structure.
</Info>

### `/components` - Reusable Components

Shared UI components and context providers used throughout the application.

#### Context Providers

<CardGroup cols={2}>
  <Card title="UserContext.tsx" icon="user">
    Global user state management
    - Stores authenticated user data
    - Provides `useUser()` hook
    - TypeScript interface for type safety
  </Card>
  <Card title="SecureStoreContext.tsx" icon="lock">
    Secure storage operations
    - `saveSecureData()` - Encrypt and store
    - `loadSecureData()` - Retrieve and decrypt
    - `deleteSecureData()` - Remove stored data
  </Card>
</CardGroup>

#### UI Components

**Themed Components**:
- **`ThemedText.tsx`** - Text component with automatic theme adaptation
- **`ThemedView.tsx`** - View container with theme-aware styling

**Specialized Components**:
- **`Collapsible.tsx`** - Expandable/collapsible content container
- **`ParallaxScrollView.tsx`** - Scroll view with parallax header effect
- **`HelloWave.tsx`** - Animated wave emoji component
- **`ExternalLink.tsx`** - Link component for external URLs

**Navigation Components**:
- **`navigation/TabBarIcon.tsx`** - Icon component for tab bar items

#### Tests

- **`__tests__/ThemedText-test.tsx`** - Unit tests for ThemedText component
- **`__tests__/__snapshots__/`** - Jest snapshot files

### `/hooks` - Custom React Hooks

Reusable logic encapsulated in custom hooks following React best practices.

| Hook | Purpose | Platform |
|------|---------|----------|
| `useColorScheme.ts` | Detect system color scheme (light/dark) | Native |
| `useColorScheme.web.ts` | Web-specific color scheme detection | Web |
| `useThemeColor.ts` | Get theme-specific color values | All |

<Note>
  The `.web.ts` extension allows platform-specific implementations. React Native automatically uses `useColorScheme.web.ts` when running on web and `useColorScheme.ts` on native platforms.
</Note>

## File Naming Conventions

<Info>
  BioTea follows these conventions:
  - **Components**: PascalCase (e.g., `UserContext.tsx`, `ThemedText.tsx`)
  - **Routes**: lowercase (e.g., `login.tsx`, `index.tsx`)
  - **Layouts**: `_layout.tsx` (special expo-router convention)
  - **Hooks**: camelCase with 'use' prefix (e.g., `useColorScheme.ts`)
  - **Constants**: PascalCase (e.g., `Colors.ts`, `Urls.ts`)
</Info>

## Import Patterns

Common import patterns used throughout the codebase:

```tsx
// React and React Native
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

// Expo Router
import { Stack, Tabs, useRouter } from 'expo-router';

// Custom Contexts

import { useSecureStore } from '../components/SecureStoreContext';

// Constants
import { Colors } from '../constants/Colors';
import { apiEndpoints } from '../constants/Urls';

// Custom Hooks
import { useThemeColor } from '../hooks/useThemeColor';
```
